# What is it?
<ul>
<Appear>
    <li>Regular CSS, with special naming conventions for clarity</li>
    <li>Stands for "Block Element Modifier"</li>
    <li>Component-oriented thinking for CSS</li>
</Appear>
</ul>

<Notes>
- Became popular in the early days of atomic design thinking, before component-based architecture on the front-end really took off
- Early example of component thinking in CSS
</Notes>

---

.<span style={{color: 'mediumaquamarine'}}>block</span><span style={{color: 'orchid'}}>__element</span><span style={{color: 'tomato'}}>--modifier</span>


<Notes>
- Always use classes
- Block level acts as a namespace for all styles
- Elements are children of the block
- Modifiers usually represent different states the block or elements can be in
</Notes>
